Core Python and Data Structures

Variables, strings, lists, dictionaries, and control flow — the foundation everything else is built on.

Mar 27 – Apr 3 ~30 hours 5 concepts 4 guided exercises 1 milestone deliverable

This week is the foundation. Every script you write going forward — every API call, every AI automation, every report generator — is built from the concepts covered here. Don't rush through it. By the end you should be able to write loops and dictionaries from memory without looking anything up.

The data in all exercises this week is modeled directly on finance operations work — consulting time entries, invoice amounts, client records. Nothing is abstract for the sake of it. Every exercise is something you could realistically encounter in a real workflow.

The goal isn't just to understand these concepts. It's to internalize them deeply enough that when you start building real automations against real systems, the Python itself is never the bottleneck. You want your mental energy going toward the problem you're solving, not the syntax of how to solve it.

Key Terms

Every term you'll encounter this week, defined in plain language. Come back here whenever something feels fuzzy. These aren't just vocabulary — they're the mental models that make everything else click.

Variable
A labeled box that stores a value. total = 24500 creates a box called total holding 24500. Use the name anywhere after that to retrieve the value.
Data type
The kind of thing stored in a variable. Python handles strings, integers, floats, and booleans differently. The type determines what operations are valid.
String
Any text value wrapped in quotes. "Accenture" is a string. "24500" is also a string — quotes make it text, not a number you can do math with.
Integer
A whole number with no decimal point. 24500 is an integer. You can add, subtract, multiply, and divide integers freely.
Float
A number with a decimal point. 24500.50 is a float. The name comes from "floating point" — how computers represent decimal numbers internally.
Boolean
Either True or False and nothing else. Used constantly in conditions. is_approved = True is a boolean. Case-sensitive — must be capitalized.
F-string
A string prefixed with f that lets you embed variables directly. f"Total: ${total:,.2f}" replaces {total} with the actual value, formatted with commas and 2 decimal places.
List
An ordered collection of items in square brackets. ["Accenture", "Deloitte"]. Items have positions starting at 0. You can add, remove, and loop through items.
Index
The position of an item in a list. Starts at 0. In ["A", "B", "C"], "A" is at index 0, "B" at index 1, "C" at index 2. Negative indexes count from the end.
Dictionary
A collection of key-value pairs in curly braces. {"client": "Accenture", "amount": 24500}. Look up values by key name, not by position. This is how API responses are structured.
Key
The label on the left side of a colon in a dictionary. Used to retrieve its paired value. Keys must be unique within a dictionary.
Value
The data on the right side of a colon in a dictionary. What you get back when you look up a key. Values can be any data type — strings, numbers, lists, even other dictionaries.
Loop
Code that repeats. A for loop runs a block of code once for each item in a collection. A while loop runs until a condition becomes false.
Iteration
One pass through a loop. Looping through 15 time entries performs 15 iterations — one per entry. The variable in the loop holds the current item during each iteration.
Conditional
Code that only runs if a condition is true. if, elif, and else are conditionals. They let your program make decisions and take different paths.
Operator
A symbol that performs an operation. + adds, - subtracts, * multiplies, / divides, % gives the remainder, == checks equality, != checks inequality.
Modulo
The % operator. Returns the remainder after division. 10 % 3 equals 1. Used to check divisibility: n % 2 == 0 means n is even.
List comprehension
A compact one-line way to build a list from another list. [x * 2 for x in numbers] creates a new list with every item doubled. Can include a filter condition too.
Append
A list method that adds one item to the end. my_list.append("new item") — the list grows by one. Called with dot notation on the list variable.
Method
A function that belongs to a specific type of object, called with dot notation. list.append() and string.split() are methods. The dot means "call this on this object."
Type conversion
Changing a value from one data type to another. int("24500") converts the string "24500" to the number 24500. Essential when reading data from files or APIs where numbers arrive as text.
The grouping pattern
A dictionary technique for summarizing data by category. Start with an empty dict, loop through data, check if the key exists, accumulate values. The most important pattern in this entire course.
Syntax
The rules of how Python code must be written. A syntax error means Python couldn't parse your code — like a grammar error in a sentence. The error message tells you exactly which line.
Comment
A line starting with # that Python ignores. Used to explain what code does. Write comments generously — code is read far more than it is written.

Concept 1 — Variables and Data Types

What they are and why they exist

Every program you will ever write needs to store information temporarily while it does its work. Variables are how you do that.

When Python runs total = 319150, it allocates a small piece of memory, stores the number 319150 there, and attaches the label total to it. Every time you write total anywhere after that, Python retrieves what's stored there.

This matters because your code needs to work with values it doesn't know in advance. You won't hardcode every invoice amount — you'll pull them from an API and store them in variables to work with. The variable name is how you refer to a piece of data throughout your code without having to repeat the value every time.

# Variables store data with a meaningful label
client_name    = "Accenture"    # string — text in quotes
invoice_amount = 24500          # integer — whole number
tax_rate       = 0.08           # float — decimal number
is_approved    = True           # boolean — True or False

# You can update what's in a variable
invoice_amount = 26000          # now it holds 26000

# You can do math with number variables
tax       = invoice_amount * tax_rate    # 2080.0
total_due = invoice_amount + tax         # 28080.0

# You can build new values from existing ones
label = client_name + " invoice"    # "Accenture invoice"

Why data types matter

Python behaves differently depending on the type of data it's working with. "24500" and 24500 look almost identical but are completely different things — one is text, one is a number. Trying to do math on text causes an error.

# This works — adding two numbers
result = 24500 + 500          # 25000

# This crashes — can't add a string and a number
result = "24500" + 500        # TypeError: can only concatenate str (not "int") to str

# This works but is probably not what you want
result = "245" + "00"         # "24500" — string concatenation, not addition

# Type conversion — switching between types
amount_string = "24500"
amount_int    = int(amount_string)      # 24500 — now you can do math
amount_float  = float("24500.50")      # 24500.5
back_to_str   = str(24500)             # "24500" — back to text

When you pull data from a CSV file or an API, numbers almost always come in as strings. Converting them before doing math is one of the most common things you'll do in real scripts. Getting this wrong silently is one of the most common bugs beginners hit.

Naming variables well

Variable names should be descriptive enough that someone reading your code understands what the variable holds without needing a comment. Use lowercase with underscores for multiple words — this is the Python standard.

# Good names
invoice_amount  = 24500
consultant_name = "Sarah Chen"
is_approved     = True
total_hours     = 47.5

# Bad names — too vague or too cryptic
x  = 24500
n  = "Sarah Chen"
t  = True
th = 47.5
Why this matters for AI automation

Every piece of data you send to an AI model or receive back passes through variables. When you call the Claude API and get a response, you store it in a variable. When you build a prompt from real invoice data, you pull values out of variables and embed them in a string. The entire automation pipeline — pull data from a source system, process it, send to an AI model, receive the response, write output somewhere useful — is a series of variables passing data from one step to the next. Variables aren't a beginner concept you'll outgrow. They're the connective tissue of every automation you'll ever build.

Concept 2 — String Formatting with F-Strings

What they are and why they exist

Raw data is ugly. 24500 means nothing to someone reading a report. $24,500.00 means something. F-strings are how you turn raw data into readable output — and more importantly, they're how you build dynamic prompts for AI models.

Put an f before the opening quote of any string and you can embed variable names directly inside it using curly braces. Python replaces the {variable_name} with the actual value when the line runs.

client = "Accenture"
amount = 24500
hours  = 47.5

# Basic embedding — variable replaces the placeholder
print(f"Client: {client}")
# Output: Client: Accenture

# Format a number with commas and 2 decimal places
print(f"Invoice amount: ${amount:,.2f}")
# Output: Invoice amount: $24,500.00

# Do math directly inside the f-string
print(f"With 8% tax: ${amount * 1.08:,.2f}")
# Output: With 8% tax: $26,460.00

# Round a number to 1 decimal place
print(f"Hours logged: {hours:.1f}")
# Output: Hours logged: 47.5

Column alignment for reports

When printing multiple rows of data you often want things to line up in columns. F-strings handle this with alignment codes.

# Left-align in a field 18 chars wide, right-align in 12 chars
print(f"{'Client':<18} {'Amount':>12}")
print(f"{'Accenture':<18} ${'24500':>11,.2f}")
print(f"{'Deloitte':<18} ${'18750':>11,.2f}")
# Output:
# Client              Amount
# Accenture            $24,500.00
# Deloitte             $18,750.00

Format codes cheat sheet

:,.2fComma separator, 2 decimal places (dollars)
:.1f1 decimal place, no comma (hours, percentages)
:dInteger, no decimal point
:<15Left-align in a field 15 characters wide
:>12Right-align in a field 12 characters wide
:^10Center in a field 10 characters wide
Why this matters for AI automation

When you build prompts for an AI model, you construct them as strings with real data embedded inside. The quality of the AI output depends directly on the quality of the prompt, and a well-structured prompt almost always involves injecting real data values into a template. Every AI automation you build will use a pattern like this:

consultant = "Sarah Chen"
hours      = 47.5
threshold  = 45

prompt = f"""
You are reviewing time entry data for a consulting firm.
{consultant} logged {hours} hours this month, which exceeds
the standard threshold of {threshold} hours.
Write a brief professional summary flagging this for review.
"""

That's an f-string. Learning to format strings well isn't just about making reports readable — it's how you communicate with AI models effectively.

Concept 3 — Lists

What they are and why they exist

Most real data isn't a single value — it's a collection. A month of invoices. A roster of consultants. A set of time entries pulled from an API. Lists let you store ordered collections and process every item using loops, rather than writing separate code for each one.

# Creating a list — square brackets, comma-separated
consultants = ["Sarah Chen", "Marcus Williams", "Jordan Lee", "Priya Patel"]
amounts     = [24500, 18750, 31200, 9800, 42100]

# Accessing items by index — STARTS AT ZERO
print(consultants[0])     # "Sarah Chen"
print(consultants[1])     # "Marcus Williams"
print(consultants[-1])    # "Priya Patel" — negative indexes from the end
print(amounts[-2])        # 9800

# Slicing — get a portion of the list
first_three = amounts[0:3]    # [24500, 18750, 31200]
last_two    = amounts[-2:]    # [9800, 42100]
middle      = amounts[1:4]    # [18750, 31200, 9800]

# Useful built-in functions on lists
print(len(amounts))     # 5 — number of items
print(sum(amounts))     # 126350 — total
print(min(amounts))     # 9800
print(max(amounts))     # 42100

# Checking membership
print("Sarah Chen" in consultants)     # True
print("Bob Smith"  in consultants)     # False

# Adding to a list
consultants.append("Ryan Torres")
print(len(consultants))    # now 5

Looping through a list

The real power of lists is that you can process every item with a loop. This is how you handle datasets of any size — whether 5 entries or 5000, the code is the same.

amounts = [24500, 18750, 31200, 9800, 42100]

# Basic loop — run once for each item
for amount in amounts:
    print(f"  ${amount:,.2f}")

# Loop with index — when you need the position too
for i, amount in enumerate(amounts):
    print(f"  Invoice {i+1}: ${amount:,.2f}")

# Loop and accumulate
total = 0
for amount in amounts:
    total = total + amount
    # shorthand: total += amount
print(f"Total: ${total:,.2f}")    # $126,350.00

# Loop with a condition — only process some items
for amount in amounts:
    if amount > 20000:
        print(f"  Large invoice: ${amount:,.2f}")

List comprehensions

A more compact way to build a new list from an existing one. The pattern is: [expression for item in list] — optionally with a condition at the end. Takes some getting used to but becomes second nature quickly.

amounts = [24500, 18750, 31200, 9800, 42100]

# Long way — build a new list with a regular loop
doubled = []
for amount in amounts:
    doubled.append(amount * 2)

# Short way — list comprehension does the exact same thing
doubled = [amount * 2 for amount in amounts]

# With a filter condition
large = [amount for amount in amounts if amount > 20000]
# [24500, 31200, 42100]

# Transforming and filtering together
large_doubled = [amount * 2 for amount in amounts if amount > 20000]
# [49000, 62400, 84200]

# Pulling a specific field from a list of dictionaries
invoices = [{"client":"Accenture","amount":24500},{"client":"Deloitte","amount":18750}]
clients  = [inv["client"] for inv in invoices]
# ["Accenture", "Deloitte"]
Why this matters for AI automation

API responses from source systems like Salesforce and Kantata come back as lists — specifically lists of dictionaries, one per record. Every time you pull time entries or invoice records from an API, you get back a list. You loop through it, filter it, and process each item. List comprehensions are how you filter large datasets down to just the records you care about before sending them to an AI model. This matters for two reasons: sending only relevant data makes the AI output more accurate, and it significantly reduces cost since AI models charge by the amount of text you send. Clean, filtered input produces better and cheaper AI output every time.

Concept 4 — Dictionaries

What they are and why they exist

A list is good when you have a collection of items in a specific order. A dictionary is good when you need to look something up by name rather than position. The distinction matters: a list gives you item number 3, a dictionary gives you the "amount" field.

When a system API returns data about an invoice, it doesn't hand you a list of raw values in some order you have to memorize. It gives you labeled data: "invoice_id": "INV-001", "client": "Accenture", "amount": 24500. That labeled structure is a dictionary — and it's the native format of virtually all API data.

# Creating a dictionary — curly braces, key: value pairs
invoice = {
    "invoice_id": "INV-001",
    "client":     "Accenture",
    "amount":     24500,
    "approved":   False
}

# Accessing values by key name
print(invoice["client"])      # "Accenture"
print(invoice["amount"])      # 24500

# Adding a new key-value pair
invoice["due_date"] = "2026-04-30"

# Updating an existing value
invoice["approved"] = True

# Checking if a key exists before accessing it
if "amount" in invoice:
    print(f"Amount: ${invoice['amount']:,.2f}")

# Getting all keys, all values, or both
print(list(invoice.keys()))     # ['invoice_id', 'client', 'amount', ...]
print(list(invoice.values()))   # ['INV-001', 'Accenture', 24500, ...]

# Looping through key-value pairs
for key, value in invoice.items():
    print(f"  {key}: {value}")

Lists of dictionaries — the most important structure

This is the data structure you'll encounter most in real work. Every API response, every CSV row, every database record comes back as a list of dictionaries — one dictionary per record, each with the same set of labeled fields.

# This is what real API data looks like
time_entries = [
    {"id": "TE-001", "consultant": "Sarah Chen",  "hours": 8.0,  "approved": True},
    {"id": "TE-002", "consultant": "Dana Kim",    "hours": 6.5,  "approved": True},
    {"id": "TE-003", "consultant": "Sarah Chen",  "hours": 11.5, "approved": False},
    {"id": "TE-004", "consultant": "Marcus Blake","hours": 7.0,  "approved": True},
]

# Access fields by name — not by position
for entry in time_entries:
    status = "approved" if entry["approved"] else "PENDING"
    print(f"  {entry['id']} | {entry['consultant']:<15} | {entry['hours']} hrs | {status}")

# Filter with a list comprehension
unapproved = [e for e in time_entries if not e["approved"]]
over_10    = [e for e in time_entries if e["hours"] > 10]

The grouping pattern — memorize this

This is the most important pattern in this entire course. Any time you want to summarize data by category — total revenue by client, hours by consultant, invoice count by project — this is the approach. Four steps, every time.

time_entries = [
    {"consultant": "Sarah Chen",   "hours": 8.0},
    {"consultant": "Dana Kim",     "hours": 6.5},
    {"consultant": "Sarah Chen",   "hours": 7.5},
    {"consultant": "Marcus Blake", "hours": 9.0},
    {"consultant": "Dana Kim",     "hours": 11.0},
]

# Step 1 — start with an empty dictionary
hours_by_consultant = {}

# Step 2 — loop through every record
for entry in time_entries:
    name = entry["consultant"]

    # Step 3 — if this key doesn't exist yet, create it at zero
    if name not in hours_by_consultant:
        hours_by_consultant[name] = 0

    # Step 4 — add the current value to the running total
    hours_by_consultant[name] = hours_by_consultant[name] + entry["hours"]

# Print results
print("Hours by consultant:")
for name, total in hours_by_consultant.items():
    print(f"  {name:<18} {total:.1f} hrs")

The pattern works for any grouping: replace "consultant" with "project", "client", "month" — whatever field you want to group by. Replace "hours" with "amount", "count", or any other value you want to accumulate. Same four steps every time.

Why this matters for AI automation

Source system API responses are dictionaries — every single one. When you call an API for time entries or invoice records, you get back a dictionary containing a list of dictionaries, one per record, each with labeled fields. The grouping pattern is how you summarize those records before passing them to an AI model. Instead of sending 200 raw time entries to Claude and asking it to figure out the totals, you group by consultant in Python first, then send a clean summary. This produces dramatically better AI output, costs a fraction of the price, and runs significantly faster. The grouping pattern also shows up in report generation, anomaly detection, and data reconciliation — the core workflows in any finance operations automation. This isn't just a Python skill. It's the fundamental building block of everything you'll build.

Concept 5 — Control Flow

If / elif / else — making decisions

Code that takes different paths depending on whether conditions are true or false. This is how your automations make decisions without a human involved.

hours = 47.5

# Basic if/elif/else
if hours > 45:
    print("Over capacity — flag for review")
elif hours < 20:
    print("Under capacity — check in with consultant")
else:
    print("Normal range — no action needed")

# Combining multiple conditions
amount = 35000
client = "Accenture"

if amount > 30000 and client == "Accenture":
    print("Large Accenture invoice — requires director approval")

if amount > 50000 or not approved:
    print("High priority — escalate immediately")

# Checking multiple values
status = "pending"
if status in ["pending", "draft", "on-hold"]:
    print("Invoice requires attention")

For loops — processing every item

consultants = ["Sarah Chen", "Dana Kim", "Marcus Blake"]

# Loop through a list
for consultant in consultants:
    print(f"Processing entries for {consultant}")

# range() generates a sequence of numbers
for i in range(5):            # 0, 1, 2, 3, 4
    print(i)

for i in range(1, 6):         # 1, 2, 3, 4, 5
    print(i)

for i in range(0, 100, 10):   # 0, 10, 20, 30 ... 90
    print(i)

# Loop through both index and value simultaneously
for i, consultant in enumerate(consultants):
    print(f"{i+1}. {consultant}")

While loops — running until a condition changes

# Keep retrying until success — common pattern in API calls
attempts = 0
success  = False

while not success and attempts < 3:
    try:
        # attempt the API call here
        success = True
    except:
        attempts += 1
        print(f"Attempt {attempts} failed, retrying...")

# Basic counter
count = 0
while count < 5:
    print(f"Count: {count}")
    count += 1    # shorthand for count = count + 1

Break and continue — controlling loop flow

entries = [8.0, 6.5, 11.5, 7.0, 14.0, 9.0]

# break — stop the loop immediately when condition is met
for hours in entries:
    if hours > 13:
        print(f"Found extreme entry: {hours} hrs — stopping search")
        break    # exits the loop entirely

# continue — skip this iteration, move to the next
for hours in entries:
    if hours < 7:
        continue    # skip entries under 7 hours
    print(f"Processing: {hours} hrs")
Why this matters for AI automation

Control flow is what transforms a script into an automation. Without it, your code does the same thing every time regardless of the data. With it, your code makes decisions: Is this time entry unusual enough to flag? Should this record be sent to the AI model for analysis or can it be skipped? If the API returns an error, should the script retry or fail gracefully? Every one of those decisions is an if statement. The loops are how you process every record in a dataset — a monthly export might have 300 time entries, and you need the same logic applied to every one without writing separate code for each. When you combine loops, conditions, and dictionaries, you have everything you need to build real automations that handle real data at real scale.

invoices = [

amounts = [inv["amount"] for inv in invoices]

Python's built-in functions handle the math. The f-string format codes make it readable.

total   = sum(amounts)
average = total / len(amounts)

maximum = max(amounts)

print("=" * 45)
print("INVOICE SUMMARY")
print("=" * 45)
print(f"{'Total invoices:':<22} {len(invoices)}")

Run it now. You should see a clean formatted summary. If you get an error, read the message — it tells you the line number and what went wrong.

Step 4 — Paid vs unpaid breakdown

A loop with a conditional. For each invoice we check the "paid" field and add the amount to the right running total.

paid_total   = 0
unpaid_total = 0

for inv in invoices:
    if inv["paid"] == True:
        paid_total   = paid_total   + inv["amount"]
    else:
        unpaid_total = unpaid_total + inv["amount"]

print()
print(f"{'Paid:':<22} ${paid_total:,.2f}")
print(f"{'Outstanding:':<22} ${unpaid_total:,.2f}")
Step 5 — Revenue by client using the grouping pattern

This is the most important pattern in the course. Four steps: empty dict, loop, check key, accumulate. The result is a dictionary where each client name maps to their total revenue.

client_totals = {}

for inv in invoices:
    client = inv["client"]
    if client not in client_totals:
        client_totals[client] = 0
    client_totals[client] = client_totals[client] + inv["amount"]

print()
print("REVENUE BY CLIENT")
print("-" * 30)
for client, total in client_totals.items():
    print(f"  {client:<14} ${total:,.2f}")
