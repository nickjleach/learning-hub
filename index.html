Core Python and Data Structures

Variables, strings, lists, dictionaries, and control flow — the foundation everything else is built on.

Mar 27 – Apr 3 ~30 hours 5 concepts 4 guided exercises 1 milestone deliverable

Key Terms

Variable
A labeled box that stores a value. total = 24500 creates a box called total holding 24500.
Data type
The kind of thing stored in a variable. Python handles strings, integers, floats, and booleans differently.
String
Any text value wrapped in quotes. "Accenture" is a string. "24500" is also a string — quotes make it text, not a number.
Integer
A whole number with no decimal. 24500 is an integer. You can do math with it.
Float
A number with a decimal point. 24500.50 is a float. From "floating point" — how computers handle decimals.
Boolean
Either True or False. Used constantly in conditions. is_paid = True is a boolean variable.
F-string
A string prefixed with f that embeds variables directly. f"Total: ${total:,.2f}" replaces {total} with the actual value.
List
An ordered collection in square brackets. ["Accenture", "Deloitte"]. Items have positions starting at 0.
Index
The position of an item in a list. Starts at 0. In ["A","B","C"], "A" is at index 0, "B" at index 1.
Dictionary
Key-value pairs in curly braces. {"client":"Accenture","amount":24500}. Look up values by key name, not position.
Loop
Code that repeats. A for loop runs once per item. A while loop runs until a condition is false.
List comprehension
A one-line way to build a list. [x*2 for x in nums] creates a new list with every item doubled.
Modulo
The % operator. Returns the remainder after division. 10 % 3 = 1. Used to check divisibility.
The grouping pattern
Empty dict → loop → check if key exists → accumulate. Used any time you want totals by client, hours by consultant, etc.
Type conversion
Changing between data types. int("24500") converts text to a number. Common when reading from files or APIs.
Syntax
The rules of how Python must be written. A syntax error means Python couldn't parse your code — like a grammar error.

Concept 1 — Variables and Data Types

Every program needs to store information temporarily. Variables are how you do that. When Python runs total = 319150 it stores that number in memory and attaches the label total to it. Every time you use that name later, Python retrieves it.

client_name    = "Accenture"   # string
invoice_amount = 24500         # integer
tax_rate       = 0.08          # float
is_paid        = True          # boolean

# Math with number variables
tax       = invoice_amount * tax_rate   # 1960.0
total_due = invoice_amount + tax        # 26460.0

# Type conversion — numbers from APIs often arrive as strings
amount_str = "24500"
amount_int = int(amount_str)    # now you can do math
amount_flt = float("24500.50")  # decimal version
Why this matters for AI

Every piece of data you send to Claude or receive back passes through variables. When you build a prompt from real invoice data, you pull values out of variables and embed them in a string. The entire automation pipeline — pull from Kantata, process, send to Claude, write output — is a chain of variables passing data from step to step.

Concept 2 — String Formatting with F-Strings

Raw data is ugly. 24500 means nothing to a manager. $24,500.00 means something. F-strings turn raw data into readable output — and they're how you build prompts for AI models.

client = "Accenture"
amount = 24500

print(f"Client: {client}")
# Output: Client: Accenture

print(f"Amount: ${amount:,.2f}")
# Output: Amount: $24,500.00

# Column alignment for reports
print(f"{'Client':<15} {'Amount':>12}")
print(f"{client:<15} ${amount:>11,.2f}")
# Client              Amount
# Accenture       $24,500.00
Why this matters for AI

Every prompt you send to Claude is a string you build with real data embedded inside it. prompt = f"Consultant {name} logged {hours} hours. Flag any concerns." — that's an f-string. The quality of your AI output depends directly on how well you construct these strings. This isn't just formatting — it's how you talk to AI models.

Concept 3 — Lists

Most real data is a collection. A month of invoices. A roster of consultants. Lists let you store ordered collections and process every item with a loop.

amounts = [24500, 18750, 31200, 9800, 42100]

# Indexing — starts at 0
print(amounts[0])     # 24500
print(amounts[-1])    # 42100 (from the end)

# Slicing
print(amounts[0:3])   # [24500, 18750, 31200]

# Loop and accumulate
total = 0
for amount in amounts:
    total = total + amount

# List comprehension — compact and powerful
large = [a for a in amounts if a > 20000]
# [24500, 31200, 42100]
Why this matters for AI

Every API response from Salesforce, Kantata, and Claude returns data as lists. You'll loop through them, filter them, and process each item. List comprehensions let you filter a dataset down to just the flagged entries before sending to Claude — smaller, cleaner input means better and cheaper AI output.

Concept 4 — Dictionaries

A dictionary stores labeled data — key-value pairs. When an API returns invoice data, it doesn't give you a list of raw values, it gives you "invoice_id": "INV-001", "client": "Accenture", "amount": 24500. That's a dictionary.

invoice = {"invoice_id":"INV-001","client":"Accenture","amount":24500,"paid":False}

print(invoice["client"])    # "Accenture"
print(invoice["amount"])    # 24500

# The grouping pattern — memorize this
invoices = [
    {"client":"Accenture","amount":24500},
    {"client":"Deloitte", "amount":18750},
    {"client":"Accenture","amount":15600},
]

client_totals = {}
for inv in invoices:
    client = inv["client"]
    if client not in client_totals:
        client_totals[client] = 0
    client_totals[client] = client_totals[client] + inv["amount"]

for client, total in client_totals.items():
    print(f"{client:<15} ${total:,.2f}")
Why this matters for AI

Salesforce and Kantata API responses are dictionaries — every single one. The grouping pattern is how you'll summarize records before sending them to Claude. Instead of passing 200 raw entries, you group by consultant, calculate totals, and send a clean summary. This makes AI output dramatically better and dramatically cheaper. Dictionaries are the prerequisite for everything in Phase 3 and beyond.

Concept 5 — Control Flow

If/elif/else makes decisions. For loops process every item. Together they're how your automations work without you.

# Decisions
hours = 47.5
if hours > 45:
    print("Over capacity — flag for review")
elif hours < 20:
    print("Under capacity — check in")
else:
    print("Normal range")

# Combining conditions
if amount > 30000 and client == "Accenture":
    print("Needs director approval")

# For loop with range
for i in range(1, 6):     # 1, 2, 3, 4, 5
    print(i)

# Break and continue
for amount in amounts:
    if amount < 10000:
        continue          # skip small invoices
    if amount > 60000:
        print("Unusually large")
        break             # stop after first hit
Why this matters for AI

Control flow is how your automations make decisions without you. Is this time entry unusual enough to flag? Should it go to Claude for analysis or be skipped? Should the script retry on an API error? Every one of those decisions is an if statement. Loops are how you process every record in a dataset — you can't write separate code for each of the 500 entries in a monthly export.

invoices = [

    {"id":"INV-005","client":"Bluewater", "amount":15300,"paid":True},
    {"id":"INV-006","client":"Redstone",  "amount":28900,"paid":True},
    {"id":"INV-007","client":"Northgate", "amount":33400,"paid":False},
    {"id":"INV-008","client":"Clearfield","amount":7200, "paid":True},
]
Step 2 — Pull amounts into a list
amounts = [inv["amount"] for inv in invoices]
Step 3 — Summary stats
total   = sum(amounts)
average = total / len(amounts)
minimum = min(amounts)
maximum = max(amounts)

print("=" * 45)
print("INVOICE SUMMARY")
print("=" * 45)
print(f"{'Total invoices:':<20} {len(invoices)}")
print(f"{'Total revenue:':<20} ${total:,.2f}")
print(f"{'Average:':<20} ${average:,.2f}")
print(f"{'Smallest:':<20} ${minimum:,.2f}")
print(f"{'Largest:':<20} ${maximum:,.2f}")
Step 4 — Paid vs unpaid
paid_total = 0
unpaid_total = 0
for inv in invoices:
    if inv["paid"]:
        paid_total   += inv["amount"]
    else:
        unpaid_total += inv["amount"]

print(f"\n{'Paid:':<20} ${paid_total:,.2f}")
print(f"{'Outstanding:':<20} ${unpaid_total:,.2f}")
Step 5 — Revenue by client
client_totals = {}
for inv in invoices:
    client = inv["client"]
    if client not in client_totals:
        client_totals[client] = 0
    client_totals[client] += inv["amount"]

print("\nREVENUE BY CLIENT")
print("-" * 30)
for client, total in client_totals.items():
    print(f"{client:<15} ${total:,.2f}")

Guided Practice — Part 2 (Hints, You Think)

Challenge 2.1 Overdue High Value Invoices ▼

Find every invoice that is unpaid AND over $20,000. Print the ID, client, and amount under a header "OVERDUE HIGH VALUE".

Hint: Loop through invoices, check two conditions with and, print with f-strings. You've seen all three already.
Challenge 2.2 Outstanding Percentage ▼

Calculate what percentage of total revenue is currently unpaid. Print it rounded to one decimal place.

Hint: Percentage = (unpaid / total) × 100. You already have both numbers. Use round(number, 1).
Challenge 2.3 Top Client by Revenue ▼

Find which client has the highest total revenue. Print their name and total. Don't hardcode the answer.

Hint: You already built client_totals. Loop through it tracking the highest value seen so far. Start with top_client = None and top_total = 0.
Challenge 2.4 Sorted Invoice List ▼

Print all invoices sorted largest to smallest. Show ID, client, amount, and paid status.

Hint: Use sorted(invoices, key=lambda x: x["amount"], reverse=True). The lambda just means "sort by the amount field."
